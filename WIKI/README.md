## Kuidas seda kausta kasutada?

1. Alusta faili **projekti-kirjeldus.md** täitmisest.
2. Seejärel tee läbi analüüsifailid: **persoonad** ja **stsenaariumid**.
3. Pane paika üldine ajakava ja sprindid.
4. Loo töö käigus issue’d vastavalt failile **planeeritavad-issue-d.md**.
5. Projekti jooksul täida vahekokkuvõtteid ja retrospektiive.
6. Projekti lõpus täida **lõppkokkuvote.md**.

## Tähtis

- See **WIKI** kaust on repositooriumi sees olev dokumentatsioonikaust.
- See **ei ole sama asi** mis GitHubi eraldi Wiki funktsioon.
- GitHub Wiki kasutamine võib olla lisavõimalus edasijõudnutele.
